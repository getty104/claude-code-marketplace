# Serena MCP エキスパートガイド

## 使用タイミング

- **プロジェクト開始時（必須）**: 初期化とオンボーディング
- **コード編集・解析時（最優先）**: ファイル全体読込の代わりにシンボル単位で操作
- **リファクタリング時**: 依存関係を分析してから変更

## 重要な原則

1. **ファイル全体読込は最終手段**: まずSerenaでシンボル検索
2. **トークン効率**: シンボル単位の操作で大幅削減
3. **Worktree使用時**: `.serena`を親からコピー（`cp -r ../../.serena .serena`）


## ツールの取得
serena mcpで使えるツールは、以下のコマンドで取得できます。

```bash
uvx --from git+https://github.com/oraios/serena serena tools list
```

## 主なワークフロー

### 1. プロジェクト初期化（初回のみ）
プロジェクトの初期化は、以下のステップで行います。
なお、アクティベートとオンボーディングの方法は「ツールの取得」を参照してください。

- ls コマンドで `.serena` ディレクトリの存在を確認
- 存在しない場合、Worktreeの場合は親から `.serena` をコピー
- serenaを使ってプロジェクトをアクティベートし、オンボーディングを実行

### 2. コード分析・編集
「ツールの取得」を実行して、コードの検索や編集に必要なツールを取得し、利用してください。
